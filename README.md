# NPO 소스 레포지토리 템플릿 (어플리케이션 코드용)

새로운 서비스를 개발하실 때 이 템플릿의 구조를 복사해서 시작하시면 됩니다!

## 🚀 파이프라인 설정 가이드

개발자 분들께서는 CI/CD 스크립트를 직접 짜실 필요가 전혀 없습니다.

1. `.github/workflows/build.yml` 파일을 엽니다.
2. 파일 내부에 있는 `# TODO:` 주석을 따라 변수 2가지(`image_name`, `config_repo`)만 본인 앱 이름에 맞게 수정해주세요.
3. 코드를 `main` 브랜치에 푸시하면 도커 빌드, Harbor 업로드, Helm 레포지토리(`npo-deploy-repo-template` 기반)의 이미지 태그 업데이트까지 **모두 플랫폼 공통 템플릿에서 자동으로 수행**합니다!

## 💡 유의 사항
- 빌드에 필요한 `Dockerfile`은 앱 특성에 맞게 직접 루트 디렉토리에 작성해주셔야 합니다.
- (플랫폼 공통 로봇 계정이 주입되어 있으므로) GitHub Repository Secret을 별도로 귀찮게 등록하실 필요가 없습니다!
