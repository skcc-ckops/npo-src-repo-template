# NPO 애플리케이션 통합 레포지토리 템플릿

새로운 서비스를 개발하실 때 무조건 이 템플릿을 복제(Clone)하여 프로젝트를 시작하시면 됩니다!
이 레포지토리 하나에서 개발과 배포 통제가 한 방에 끝납니다.

---

## 🚀 온보딩 가이드 (가장 먼저 해야 할 일)

개발팀은 파이프라인 스크립트를 짜거나 헬름(Helm)의 복잡한 ALB, 리소스 셋팅을 건드릴 필요가 전혀 없습니다. 
프로젝트 안에 **`TODO: (필수)`** 라고 적힌 주석을 찾아 본인의 앱 이름으로만 채워 넣으시면 모든 배포 준비가 끝납니다!

### ✅ 1. GitHub Actions 워크플로우 명시
- **위치:** `.github/workflows/build.yml`
- `TODO: (필수)` 를 찾아 `image_name` 값을 본인의 애플리케이션 이름(예: `my-app`)으로 수정해주세요.

### ✅ 2. Helm Chart 기본 정보 수정
- **위치:** `helm/Chart.yaml`
- `TODO: (필수)` 를 찾아 `name` 과 `description` 값을 본인의 애플리케이션 이름으로 수정해주세요.

### ✅ 3. Helm Values 배포 환경변수 설정
- **위치:** `helm/values.yaml`
- `image.repository`를 찾아 끝의 앱 이름을 수정해주세요 (예: `harbor.biz-think.net/npo/my-app`)
- URL 접속 라우팅에 사용될 `ingress` 의 `path` 를 알맞게 지정해주세요 (예: `/my-app`)
- (필수 아님) `env` 나 `resources` 도 필요하다면 추가/수정 가능합니다.

> [!TIP]
> 이제 개발을 마치고 코드를 `main` 브랜치에 푸시(Push)해 보세요!
> 별도의 인프라 레포지토리를 직접 관리할 필요 없이, 공통 파이프라인 봇이 동시성 큐(Queue) 통제를 통해 **안전하게 도커 빌드부터 ArgoCD 배포까지 알아서 자동으로 수행합니다.**

---

## 💡 유의 사항
1. **Dockerfile:** 소스코드 최상단 경로에 각 앱의 특성에 맞는 `Dockerfile`을 직접 작성해 두셔야 빌드가 정상 동작합니다.
2. **시크릿 토큰 불필요:** 템플릿 내에 플랫폼 공통 로봇 계정이 모두 연동되어 있으므로, GitHub Repository `Settings > Secrets` 탭은 아예 들어가시거나 만지실 필요가 없습니다!
